1) Domain Notes — Failure Modes & Extraction Strategy Decision Tree

What this section proves: (1) empirical failure modes across four classes (A/B/C/D), (2) an explicit decision tree grounded in measurable signals, and (3) a pipeline diagram showing where quality checks occur.

1.1 Digital vs. Scanned: observable criteria (from our corpus)

Native digital indicators
  • Non-zero avg_char_density and high font_coverage (≈1.0).
  • Text stream extractable: pages produce non-empty text without OCR.
  • Often lower image dominance (avg_image_density is moderate/low).
Example: tax_expenditure_ethiopia_2021_22 is native_digital with strong char density + font coverage.
Scanned indicators
  • Near-zero avg_char_density and low/zero font_coverage.
  • High avg_image_density (often near 1.0): pages are raster images.
  • Text extraction yields empty strings; OCR/VLM required.
Example: 2013-E.C-Procurement-information is scanned_image with avg_char_density≈0, avg_image_density≈1.

1.2 Failure modes by document class (A/B/C/D) — empirical + technical causes

Class A — Native financial Examples: Annual_Report_JUNE-2023, tax_expenditure_ethiopia_2021_22
  • Multi-column reading-order collapse: annual reports often use two columns. Naive extraction can interleave columns, breaking sentence continuity and LDU boundaries. Root cause: insufficient layout modeling.
  • Table semantics loss: financial statements embed structured tables; text-only extraction removes cell boundaries (headers/rows). Root cause: plain text output lacks table normalization.
Mitigation: Route native_digital + multi_column to Strategy B. Escalate A→B if Strategy A confidence 0.8.
Measured signals:
  • Annual_Report_JUNE-2023 — native_digital / multi_column — char_density=0.005930, image_density=0.482, table_density=0.213, font_coverage=1.00
  • tax_expenditure_ethiopia_2021_22 — native_digital / single_column — char_density=0.003785, image_density=0.022, table_density=0.000, font_coverage=1.00
Class B — Scanned legal / procedural Examples: 2013-E.C-Procurement-information, Security_Vulnerability_Disclosure_Standard_Procedure_1
  • Zero-text extraction: scanned PDFs carry no character stream; fast extractors return empty content. Root cause: raster-only pages.
  • Clause/list boundary loss: procedural docs use numbering/indentation; OCR may merge lines or drop punctuation, corrupting downstream chunking.
Mitigation: Route scanned to Strategy C (Vision). Budget cap is enforced; non-budget failures degrade gracefully with metadata.degraded=true (no silent pass).
Measured signals:
  • 2013-E.C-Procurement-information — scanned_image / figure_heavy — char_density=0.000000, image_density=1.000, table_density=0.000, font_coverage=0.00
  • Security_Vulnerability_Disclosure_Standard_Procedure_1 — scanned_image / figure_heavy — char_density=0.000000, image_density=1.000, table_density=0.000, font_coverage=0.00
  • Audit Report - 2023 — scanned_image / figure_heavy — char_density=0.000023, image_density=0.902, table_density=0.000, font_coverage=0.10
Class C — Mixed assessment / mixed-mode Examples: CBE ANNUAL REPORT 2023-24, fta_performance_survey_final_report_2022
  • Document-level classification hides page variance: mixed docs combine digital text pages and image-heavy pages (scanned inserts / full-page figures). Root cause: per-page origin differs.
  • Form-fillable semantics: some docs include annotation layers; pure text extraction can miss user-facing form semantics. Root cause: content split across layers.
Mitigation: Start with Strategy B for mixed; escalate to Strategy C when Strategy B confidence 0.7 or content coverage is insufficient.
Measured signals:
  • CBE ANNUAL REPORT 2023-24 — mixed / figure_heavy — char_density=0.001050, image_density=0.556, table_density=0.000, font_coverage=0.80
  • fta_performance_survey_final_report_2022 — form_fillable / multi_column — char_density=0.005237, image_density=0.007, table_density=0.040, font_coverage=1.00
  • 20191010_Pharmaceutical-Manufacturing-Opportunites-in-Ethiopia_VF — native_digital / multi_column — char_density=0.000737, image_density=0.449, table_density=1.019, font_coverage=1.00
Class D — Table-heavy fiscal Examples: Consumer Price Index August 2025
  • Dense table boundary ambiguity: CPI/statistical tables may include merged headers and multi-line cells; heuristic table detectors may under/over-segment. Root cause: table structure not explicit in PDF stream.
  • Provenance ambiguity when flattened: without cell-level bboxes, citations become non-auditable. Root cause: losing structured table objects.
Mitigation: Prefer Strategy B for native table-heavy docs; prefer Strategy C for scanned/form-fillable. Preserve tables using TableCell objects with bboxes.
Measured signals:
  • Consumer Price Index August 2025 — form_fillable / table_heavy — char_density=0.002901, image_density=0.129, table_density=0.749, font_coverage=1.00

3) Cost Analysis — Per-document estimates by strategy tier

Numerical estimates (config-driven)

  • Tier A (Fast Text): $0.00 / page
  • Tier B (Layout-Aware): $0.01 / page
  • Tier C (Vision): $0.50 / page
  • Budget cap (Vision): $5.00 / document
These values live in rules.yaml so budgeting can be tuned without code changes.

Derivation transparency (explicit chain)

  • Tier A/B monetary cost: treated as ~$0 (local compute) for interim; we still track processing time empirically via the ledger.
  • Tier C monetary cost: we model an effective VLM price via:
    • estimated tokens/page = 700
    • assumed effective price (vision+image overhead) ≈ $0.70 / 1K tokens
    • cost/page ≈ tokens/page ÷ 1000 × price ≈ 700/1000×0.70 ≈ $0.49 ≈ configured $0.50
  • Processing time uses observed ledger time for processed pages, and projects linearly to full document length for planning.
  • Cost varies across classes via (a) page count and (b) routing: scanned/table-heavy docs trigger higher tiers.
Quality ↔ cost link: Tier B preserves tables/figures/reading order; Tier C recovers text from scanned pages when A/B fail.

3.2 What higher-cost tiers buy us (tied to document types)

  • A → B: moves from plain text stream to layout-preserved blocks with bounding boxes, plus structured tables and figures. Critical for Class A (multi-column financial) and Class D (table-heavy fiscal).
  • B → C: recovers content when the PDF has no text stream (scanned images / low font coverage). Critical for Class B scanned procedural docs and mixed-mode inserts in Class C.
  • Budget control: Tier C enforces budget_cap_per_doc and now degrades gracefully on non-budget failures (so the pipeline never silently returns empty output).
Quality gate: If final confidence remains below 0.8, we flag metadata.review_required=true rather than silently passing low-fidelity extraction.
